# PDF导入功能设计文档

## 概述

为Word2Article应用添加PDF导入功能，支持从"不背单词"app导出的PDF中提取单词，提供自动解析和手动选择两种模式。

## 目标

1. 支持上传PDF文件
2. 自动识别并提取PDF中的英文单词
3. 提供手动选择单词的界面
4. 将提取的单词导入到学习流程中

## 技术方案

### 客户端PDF解析

使用 `pdfjs-dist` 库在浏览器端解析PDF，确保用户隐私（文件不离开浏览器）。

### 核心功能

#### 1. 文件上传
- 支持拖拽上传和点击选择
- 文件大小限制：10MB
- 仅支持 `.pdf` 格式

#### 2. 自动解析模式
- 解析PDF文本内容
- 使用正则表达式提取英文单词
- 过滤噪音（页码、标点、数字等）
- 显示提取的单词列表供用户确认

#### 3. 手动选择模式
- 显示PDF文本内容
- 用户可以点选单词
- 支持多选和批量操作

#### 4. 导入流程
```
上传PDF → 选择模式 → 预览/编辑 → 确认导入 → 进入学习
```

## 组件设计

### PdfImport组件
- 位置：`components/PdfImport.tsx`
- 功能：文件上传、模式切换、预览、确认
- 状态管理：
  - `file`: 上传的文件
  - `mode`: 'auto' | 'manual'
  - `words`: 提取的单词列表
  - `selectedWords`: 用户选择的单词

### 首页集成
- 在首页添加"从PDF导入"按钮
- 点击后打开PdfImport组件

## 依赖

- `pdfjs-dist`: PDF解析库
- 现有依赖：Next.js, React, Tailwind CSS

## 验收标准

1. [ ] 能够上传PDF文件
2. [ ] 自动解析模式能正确提取单词
3. [ ] 手动选择模式能正常工作
4. [ ] 导入的单词能进入学习流程
5. [ ] 界面响应式，支持移动端

## 风险与限制

1. PDF格式复杂，可能影响解析准确性
2. "不背单词"导出格式可能变化
3. 大文件解析可能较慢

## 后续优化

1. 支持更多PDF格式
2. 添加单词去重功能
3. 支持批量导入
4. 添加导入历史记录
