# Word2Article — 移动端 UI 描述文档

> 用于 Google Stitch 生成新设计的参考文档

## 项目概述

Word2Article 是一款英语单词学习工具。用户输入单词列表，系统生成一篇包含这些单词的英文文章，用户通过阅读语境来记忆单词。

核心理念：**快速整理单词 + 语境阅读记忆**

目标用户：在其他背单词 App（不背单词、墨墨等）已经背过单词，想通过阅读来巩固记忆的用户。

---

## 页面清单（共 6 个页面）

### 1. 首页（记忆工作台）— `/`

**用途**：输入单词、生成文章的入口

**布局**：
- 顶部：标题「记忆工作台」+ 副标题「将零碎的单词变成文章，通过语境记忆单词」
- 中间：大文本框（贴入单词列表，支持任意格式如逗号分隔、换行分隔）
- 文本框下方：风格选择按钮组（故事 / 新闻 / 科普 / 对话）
- 底部左侧：难度切换（CET-4 / CET-6）
- 底部右侧：「一键生成」主按钮（emerald 绿色）
- 分割线
- 底部：「从 PDF 词单导入单词」按钮

**交互**：
- 用户粘贴单词 → 选择风格和难度 → 点击一键生成 → 跳转到文章页
- PDF 导入会弹出一个底部弹窗（Bottom Sheet），支持拖拽上传

**移动端特点**：
- 单列布局，max-width 672px 居中
- 文本框高度 128px
- 按钮全宽

---

### 2. 文章阅读页 — `/article/[id]`

**用途**：沉浸式阅读英文文章，点击单词查看释义

**布局**：
- 顶部：返回首页链接
- 文章卡片：
  - 顶栏：左侧「沉浸式阅读语境」标签 + 右侧功能按钮组（遮罩开关、复制全文、难度标签、风格标签）
  - 正文：英文文章内容，目标单词高亮显示
  - 单词变形对照区：两列表格，显示原形 → 变形 + 类型标签
  - 中文翻译区：折叠的参考译文
- 底部：操作链接（开始测试 / 生成新文章 / 历史记录）

**交互**：
- 点击高亮单词 → 弹出词典弹窗（Popover），显示音标、中文释义、英文释义、例句、收藏按钮
- 点击变形单词 → 弹出弹窗，额外显示变形关系（如 studies → study，名词复数）
- 遮罩模式：目标单词只显示首字母，点击揭示 1.5 秒后自动隐藏
- 非目标单词也可点击查看释义

**移动端特点**：
- 文章字号 16.5px，行高宽松
- 单词弹窗 220px 宽，自动定位避免溢出屏幕
- 变形对照区单列显示

---

### 3. 测验页 — `/quiz/[id]`

**用途**：对文章中的单词进行选择题测试

**布局**：
- 顶部：返回文章链接 + 进度指示（第 N / 总数题）
- 中间卡片：
  - 大字显示当前单词
  - 音标
  - 提示文字「选择正确的中文释义」
  - 4 个选项按钮（A/B/C/D）
- 结果区：答对/答错提示 + 下一题按钮
- 完成页：大字显示正确率百分比 + 统计 + 两个按钮（再测一次 / 返回文章）

**交互**：
- 点击选项 → 高亮正确/错误 → 显示结果 → 点击下一题
- 完成后可重测或返回文章

**移动端特点**：
- 选项按钮全宽、大点击区域（padding 16px）
- 完成页居中显示大数字

---

### 4. 生词本页 — `/favorites`

**用途**：查看收藏的单词

**布局**：
- 顶部：标题「生词本」+ 总数统计
- 单词列表：每项包含
  - 单词（emerald 色）
  - 发音按钮（喇叭图标）
  - 音标
  - 中文释义
  - 英文释义
  - 取消收藏按钮（心形图标）
- 空状态：心形图标 + 「暂无收藏单词」+ 去生成文章按钮

**移动端特点**：
- 单列卡片布局
- 每项卡片有 glass-card 毛玻璃效果

---

### 5. 历史记录页 — `/history`

**用途**：查看已生成的所有文章

**布局**：
- 顶部：标题「复习历史」+ 副标题
- 文章列表：每项包含
  - 难度标签（CET-4/CET-6）
  - 风格标签（故事/新闻/科普/对话）
  - 创建时间
  - 文章标题
  - 单词数量
  - 删除按钮（垃圾桶图标）
- 空状态：时钟图标 + 「暂无历史记录」+ 去生成文章按钮

**交互**：
- 点击卡片 → 跳转到文章详情
- 点击删除 → 确认弹窗 → 删除

**移动端特点**：
- 单列卡片布局
- 卡片可点击跳转

---

### 6. PDF 导入弹窗（底部弹窗）— 首页弹出

**用途**：从 PDF 词单中智能识别单词

**布局**：
- 顶栏：标题「从 PDF 智能识别单词」+ 关闭按钮
- 拖拽上传区：虚线边框区域，支持拖拽或点击上传
- 解析结果：标签组显示识别到的单词，可逐个点击选中/取消
- 底栏：取消 + 确认导入按钮

**移动端特点**：
- 底部弹窗样式（从屏幕底部滑出）
- 最大高度 85vh，内容可滚动
- 圆角顶部

---

## 导航结构

- **顶部导航栏**：固定在顶部，左侧 Logo + 「Word2Article」，右侧三个链接（首页 / 生词本 / 历史记录）
- **页脚**：版权信息「© 2026 Word2Article. 融入语境，高效记单词」

---

## 设计风格

- **主题**：深色为主（暗蓝黑色背景 #03060c），浅色模式也支持
- **主色调**：emerald 绿 (#10b981)，用于品牌色、按钮、高亮
- **辅助色**：蓝色 (#3b82f6)，用于变形单词高亮
- **卡片效果**：毛玻璃（backdrop-filter: blur），带微光边框和鼠标追光
- **背景**：带噪点纹理 + 三个彩色光球动画（emerald、blue、purple）
- **字体**：Inter（英文）+ 系统中文字体
- **圆角**：大圆角（rounded-xl = 12px, rounded-2xl = 16px）
- **阴影**：深色大阴影，营造悬浮感

## 关键组件说明

### WordInput（单词输入）
- 多行文本框，支持粘贴任意格式单词
- 清空按钮（出现在文本框右上角）
- 风格选择：4 个横向排列的 pill 按钮
- 难度切换：CET-4 / CET-6 分段控制器
- 生成按钮：emerald 绿色，带加载动画

### ArticleView（文章视图）
- 将英文文章中的目标单词高亮（emerald 色底 + 虚线下划线）
- 变形单词蓝色高亮
- 遮罩模式：单词显示为首字母 + 透明文字（保留宽度）

### WordPopup（单词弹窗）
- 浮动弹窗，定位在点击位置下方
- 显示：单词、发音按钮、收藏按钮、音标、中文释义、英文释义、例句
- 变形单词额外显示：变形关系 + 类型标签
- 加载状态：旋转动画
- 错误状态：错误提示

### Toast（通知）
- 类似 iOS 灵动岛的顶部通知条
- 从上方滑入，自动消失

## 移动端页面流程

```
首页（输入单词）
  ├── 一键生成 → 文章阅读页
  │     ├── 点击单词 → WordPopup 弹窗
  │     ├── 开始测试 → 测验页
  │     ├── 生成新文章 → 回到首页
  │     └── 历史记录 → 历史记录页
  ├── 从 PDF 导入 → PDF 弹窗 → 回到首页（填入单词）
  ├── 导航栏「生词本」→ 生词本页
  └── 导航栏「历史记录」→ 历史记录页
        └── 点击文章 → 文章阅读页
```

---

## 每个页面需要重新设计的要素

1. **首页**：单词输入体验、风格/难度选择器、生成按钮的视觉权重
2. **文章页**：阅读体验、单词高亮样式、弹窗交互、遮罩模式的视觉反馈
3. **测验页**：选项卡片的触感、正确/错误的视觉反馈、完成页的成就感
4. **生词本**：单词卡片的信息密度、发音/收藏的操作便捷性
5. **历史记录**：文章卡片的信息层级、删除操作的确认方式
6. **PDF 弹窗**：上传体验、单词选择的交互方式

---

## 技术约束

- 使用 Next.js App Router，页面都是 React Client Components
- 样式用 Tailwind CSS 4 + CSS 变量实现主题切换
- 移动端优先（max-width: 672px 居中）
- 需要支持深色/浅色两套主题
- 导航栏固定在顶部（sticky）
- 弹窗/Popover 需要考虑移动端的触摸交互
